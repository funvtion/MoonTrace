# Contributing

MoonTrace is a proposal-stage MoonBit project. Contributions should keep the API
small, readable, and easy to verify.

## Local Checks

Before sending changes, run moon check and moon test. If a change updates public
behavior, also update README.md or the relevant document in docs.

## Change Style

- Keep changes focused.
- Prefer explicit data structures over hidden global state.
- Add tests for behavior changes.
- Keep examples short enough for reviewers to run quickly.

## Review Focus

During the competition period, review should prioritize correctness, API
clarity, documentation accuracy, and whether the roadmap remains achievable.
